Was kann conda?

  • binäre Python-Pakete installieren
  • Environments verwalten
  • Pakete erzeugen
  • andere Programmiersprachen installieren (R)

Woher bekomme ich conda?

Python-Distributionen von Continuum Analytics

Miniconda

minimalistisches Python mit conda, pip und setuptools

Anaconda

Umfangreiche Python-Distribution mit ca. 200 meist wissenschaftlichen Paketen (numpy, scipy, pandas)

Woher kommen die Pakete?

  1. Installation mit conda: „channels“
    • Continuum Analytics conda repository (default)
    • Anaconda Cloud: private und öffentliche Repos für Pakete, Environments und Jupyter Notebooks
    • lokaler Channel (Firmennetzwerk, etc.)
  2. Installation mit pip:
    • PyPI —Python Package Index

Wo landen die Pakete?

  • im „root“ environment, z.B. 
    /home/klaus/anaconda3/python3.5/site-packages
  • im aktiven environment

conda hebt alle Versionen aller installierten Pakete (tarballs und ausgepackte Pakete) in $CONDA_BASE/pkgs auf!
⇒ aufräumen mit conda clean -pt

conda in der Praxis

Informationen ermitteln

conda info
Informationen zur Installation
conda list
installierte Pakete des aktiven Environments anzeigen
conda env list
conda info --envs
alle Environments anzeigen

conda in der Praxis

Environment anlegen

conda create --name meetup27 requests python=2.7
erzeugt das Environment meetup27
installiert Python 2.7
installiert das Paket requests
conda create -n meetup26 python=2.6
erzeugt das Environment meetup26
installiert Python 2.6
installiert keine weiteren Pakete

conda in der Praxis

Environment aktivieren

source activate envname
aktiviert envname

Environment deaktivieren

source deactivate envname
deaktiviert envname

Environment löschen

conda env remove --name envname
löscht envname

conda in der Praxis

Pakete installieren

conda install package
installiert package aus dem conda Repository in das aktuelle Environment
conda install -c dingens zeug=2016.07
installiert das Paket zeug in der Version 2016.07 aus channel dingens in der Anaconda Cloud in das aktuelle Environment
pip install package
installiert das Paket package aus PyPI

conda in der Praxis

Pakete deinstallieren

conda remove package
entfernt package aus aktuellen Environment
pip uninstall package
entfernt das mit pip installierte Paket package

conda in der Praxis

Pakete aktualisieren

conda update package
aktualisiert package
conda update --all
aktualisiert alle Pakete im aktiven Environment

conda in der Praxis

Aufräumen

Alle tarballs und ausgepackten Pakete bleiben in $CONDA_BASE/pkgs erhalten, deshalb:

conda clean -p
löscht die Pakete
conda clean -t
löscht die tarballs
conda clean --all --dry-run
zeigt, was gelöscht werden würde ;-)
